# Benchmarking — Crav Red Order Board System

## Por Que Este DS Converte Melhor

| Fator | Otimização |
|---|---|
| Anatomia | menu-board commerce cria uma experiência própria em vez de template genérico. |
| Navegação | sticky order rail with category buttons reduz fricção no caminho principal. |
| Hero | menu board hero with burger stack, price chips and immediate order action prova a proposta já na primeira dobra. |
| Componentes | menu-price-chip, combo-builder-card, sauce-selector, limited-drop-badge, delivery-toggle, receipt-summary dão vocabulário visual exclusivo. |
| Motion | snappy menu flip, sauce hover splash, cart rail lock reforça o comportamento do segmento. |

## Performance

- LCP: menor que 2.5s; hero precisa usar imagem otimizada ou CSS leve.
- INP: menor que 200ms; interações de menu-price-chip devem ser simples.
- CLS: menor que 0.1; reservar dimensões de mídia e módulos.

## SEO

- Título com segmento + diferencial real.
- Schema.org adequado ao negócio final.
- FAQs somente quando houver perguntas úteis.
- Conteúdo de Hero order board, Best sellers, Build your combo, Limited drop, Pickup/delivery choice, Sauce selector, Social proof wall, Footer as receipt deve virar heading hierarchy, não apenas blocos visuais.

## Casos Ideais

- Hamburgueria / ordering
- mobile users choosing food fast

## Riscos Evitados

- avoid restaurant brochure sections before ordering
- Repetir grid, hero ou card de outro DS.
- Usar referência como molde literal.


## Comparação Contra Template Genérico

| Camada | Decisão específica deste DS | Como validar |
|---|---|---|
| Layout | menu-board commerce | A primeira dobra não pode encaixar em outro DS sem perder sentido. |
| Navegação | sticky order rail with category buttons | O menu precisa carregar uma função do segmento. |
| Hero | menu board hero with burger stack, price chips and immediate order action | A mídia/diagrama precisa provar a promessa. |
| Motion | snappy menu flip, sauce hover splash, cart rail lock | Movimento deve reforçar comportamento, não decorar. |
| Responsivo | thumb-first category rail and cart summary | Mobile preserva conversão e leitura. |
